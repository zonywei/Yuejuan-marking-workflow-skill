# Publish Checklist

Use this checklist before promoting the repository on GitHub, skill directories, or social platforms.

## GitHub Homepage

- [ ] README has a one-sentence positioning statement.
- [ ] README explains that this is AI-assisted, teacher-confirmed, reviewable marking.
- [ ] README shows a 30-second table: input, output, value, scenario, boundaries.
- [ ] README links to a concrete demo example.
- [ ] README has installation commands.
- [ ] README has safety and privacy notes.
- [ ] README has License information.

## Demo And Examples

- [ ] The repository contains a demo under `examples/`.
- [ ] Demo data is fictional or anonymized.
- [ ] Demo includes question, answer key, rubric, model prompt, sample outputs, review queue, and teacher report.
- [ ] Demo does not include real student data, cookies, tokens, images, ledgers, or platform exports.
- [ ] If screenshots are added later, they must be synthetic, anonymized, or generated from sample data.

## Release Readiness

- [ ] `SKILL.md` validates as a skill.
- [ ] Helper scripts compile.
- [ ] `.gitignore` excludes local config and temporary grading output.
- [ ] No secrets or real student data appear in `git status` or staged files.
- [ ] Quickstart is honest. Do not claim one-click platform automation unless it exists.
- [ ] Published listings point to the current GitHub repository.
